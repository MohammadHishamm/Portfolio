## Overview

Drove a focused SEO initiative across the Hotdesk web application, addressing both technical SEO fundamentals and content-level metadata to improve crawlability and ranking in organic search results.

## Key Improvements

- **Metadata & Open Graph**: Audited and updated `<title>`, `<meta description>`, and Open Graph tags across all key pages to ensure accurate, keyword-relevant previews in search results and social shares.
- **Structured Data**: Added JSON-LD schema markup (Organization, WebPage, BreadcrumbList) to help search engines understand page context and enable rich result features.
- **Canonical URLs**: Implemented canonical tags to prevent duplicate content issues caused by query parameters and route variants.
- **Sitemap & Robots**: Reviewed and updated the XML sitemap and `robots.txt` to ensure all indexable pages were correctly included and no important routes were blocked.
- **Performance Basics**: Addressed Core Web Vitals issues (LCP, CLS) that were indirectly affecting search ranking — including lazy-loading images and fixing layout shifts.

## Technologies Used

- **Next.js** – server-side metadata, sitemap generation, and head management
- **TypeScript** – typed metadata utilities
- **JSON-LD** – structured data schema markup

## Challenges and Learnings

SEO improvements are rarely visible immediately. Setting up baseline tracking in Google Search Console before making changes was critical for measuring impact. The most impactful single fix was correcting duplicate title tags across paginated routes — a subtle issue that was suppressing ranking on several high-value pages.

## Outcome

Improvements are live across the Hotdesk web application, with measurable gains in crawl coverage and indexed pages tracked via Google Search Console.
